7. 주소 체계

7.1 Unified Address (Sapling + Orchard)

Software 계정의 기본 주소는 Sapling 과 Orchard receiver 만 포함하는 ZIP-316 Unified Address 이다 (transparent 제외). zodl/Zashi 동작과 일치한다.

  • 메인넷 접두사: u1...
  • Receivers: Orchard (Pallas 곡선) + Sapling (Jubjub 곡선)
  • 보내는 월렛이 자동으로 지원하는 가장 높은 프라이버시 레벨의 receiver 를 선택

Keystone 하드웨어 계정 (Orchard-only UA): Keystone 3 펌웨어는 UFVK 의 Sapling 부분을 export 하지 않는다. 그래서 하드웨어 계정은 orchard_address_request (Orchard Require, Sapling Omit, Transparent Omit) 로 주소를 파생한다 — 결과 UA 는 Orchard receiver 하나만 포함한다. 여전히 유효한 Zcash 주소이지만 Sapling-only 송금 지갑 (구형) 에서는 보낼 수 없다. 자세한 것은 §9.2 UFVK import.

7.2 Transparent Address

getTransparentAddress()를 통해 별도의 transparent 전용 주소를 조회할 수 있다. 이것은 동일한 seed에서 BIP-44 경로 m/44'/133'/0'로 유도된 표준 t-address (P2PKH, 메인넷 접두사 t1)이다.

7.3 zodl/Zashi 호환성

같은 니모닉으로 생성해도 우리 월렛과 zodl에서 다른 주소가 생성된다. 이것은 예상된 동작이다 — zodl의 Android SDK가 [patch.crates-io]를 통해 crates.io 공개 버전과 다른 특정 git revision의 orchard와 sapling-crypto를 사용하기 때문이다. 키 유도 수학이 다른 곡선 포인트를 생성한다. 두 주소 모두 동일한 seed에서 유도된 유효한 Zcash 주소이다.

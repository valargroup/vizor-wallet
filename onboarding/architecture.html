1.1 레이어 분리

월렛은 세 개의 레이어로 분리되어 있다:

LayerLanguage역할주요 파일
Rust Core Rust 모든 Zcash 암호학: 키 유도(ZIP-32), 주소 인코딩(ZIP-316), 블록 스캐닝(trial decryption), 월렛 DB(SQLite), commitment tree 관리. Single DB connection reused across entire sync. Progress uses WalletSummary::progress() — note-based ratio (notes scanned / total notes) that works correctly even when blocks are scanned out of order. All sync logs include [Xs] elapsed time. Errors logged via log crate → Flutter console. rust/src/wallet/, rust/src/api/
FFI Bridge Generated flutter_rust_bridge v2.11.1 codegen. rust/src/api/의 pub fn을 Dart async 함수로 변환. 메모리, 직렬화, 에러 전파를 처리. lib/src/rust/ (자동 생성), flutter_rust_bridge.yaml
C FFI C / Swift iOS background sync용. ffi.rs가 #[no_mangle] pub extern "C"로 Rust sync를 노출. Swift가 zcash_sync.h를 통해 호출. FRB와 별개의 경로. rust/src/ffi.rs, ios/Runner/zcash_sync.h, ios/Runner/BackgroundSyncManager.swift
Dart UI Dart / Flutter UI 렌더링, 상태 관리(Riverpod), 네비게이션(GoRouter), 보안 저장소(iOS Keychain / Android Keystore). Sync는 Rust에서 실행되며 Dart는 Stream으로 진행률을 수신. lib/src/providers/, lib/src/features/, lib/app.dart

1.3 왜 Rust + Flutter인가

공식 Zcash 생태계에서 Swift(iOS)와 Kotlin(Android) 모바일 SDK를 제공하지만, 둘 다 내부적으로 동일한 librustzcash Rust 크레이트를 UniFFI / JNI로 래핑한 것이다. 공식 Flutter/Dart SDK는 존재하지 않는다.

우리 접근법은 librustzcash를 FFI로 직접 호출하며, 이를 통해:

  • 단일 Rust 코드베이스로 iOS와 Android 모두 커버 — 플랫폼별 암호화 코드 불필요
  • zcash_client_backend와 zcash_client_sqlite에 중간 SDK 없이 직접 접근
  • flutter_rust_bridge가 크로스 컴파일, async, 타입 마샬링을 자동으로 처리

6.3 Send 함수 (2단계)

전송은 propose → execute 2단계로 분리되어, Sapling proving parameters가 필요한지 사전에 확인할 수 있다.

Dart FunctionRust Function설명
proposeSend({dbPath, network, toAddress, amountZatoshi, memo?}) propose_send(...) 전송 제안. ProposalResult { proposalId, needsSaplingParams, feeZatoshi } 반환. Proof 생성 없음, params 불필요.
executeProposal({dbPath, proposalId, seed, spendParamsPath?, outputParamsPath?}) execute_proposal(...) 제안 실행. zk-SNARK proof 생성 + 서명. needsSaplingParams가 true였을 경우에만 params 경로 필요.

Sapling Proving Parameters: Sapling proof 생성에 sapling-spend.params (~47MB) + sapling-output.params (~3.5MB) 파일이 필요하다. Orchard (Halo 2)는 trusted setup이 없어 파라미터 파일이 불필요하다. propose_send()의 needsSaplingParams 플래그로 필요 여부를 사전에 확인한 후, 필요 시 https://download.z.cash/downloads/에서 다운로드하고 SHA-1 검증 후 저장한다. 한 번만 다운로드하면 이후에는 로컬 파일을 사용한다.

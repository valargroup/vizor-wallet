6.3 Send 함수 (3단계: estimate → propose → execute)

전송은 estimate → propose → execute로 분리된다. 모든 함수가 account_uuid를 받아 계정별로 동작한다. ZEC 금액은 integer-only 파싱으로 zatoshi로 변환 (부동소수점 사용하지 않음).

Dart FunctionRust Function설명
estimateFee({dbPath, network, accountUuid, toAddress, amountZatoshi, memo?}) estimate_fee(...) 수수료 추정. proposal 저장 없이 수수료만 반환 (zatoshi). 전송 전 사용자에게 표시.
proposeSend({dbPath, network, accountUuid, toAddress, amountZatoshi, memo?}) propose_send(...) 전송 제안. ProposalResult { proposalId, needsSaplingParams, feeZatoshi } 반환. 내부적으로 StoredProposal { proposal, network, account_id }에 계정 ID 포함하여 메모리에 저장.
executeProposal({dbPath, lightwalletdUrl, proposalId, seed, spendParamsPath?, outputParamsPath?}) execute_proposal(...) 제안 실행. zk-SNARK proof 생성 + 서명 + gRPC broadcast. Orchard-only TX에는 no-op Sapling prover 사용 (params 불필요). 니모닉은 AccountProvider.getActiveMnemonic()에서 조회하여 seed 유도. 친화적 에러 메시지 반환.

Sapling Proving Parameters: Sapling proof 생성에 sapling-spend.params (~47MB) + sapling-output.params (~3.5MB) 파일이 필요하다. Orchard (Halo 2)는 trusted setup이 없어 파라미터 파일이 불필요하다. propose_send()의 needsSaplingParams 플래그로 필요 여부를 사전에 확인한 후, 필요 시 https://download.z.cash/downloads/에서 다운로드하고 SHA-1 검증 후 저장한다. 한 번만 다운로드하면 이후에는 로컬 파일을 사용한다. Orchard-only TX (needsSaplingParams=false)에서는 no-op Sapling prover가 사용되어 params 파일이 전혀 필요 없다.

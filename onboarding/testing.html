wallet::sync::tests (3개) — PROPOSAL_STORE 라이프사이클

Test검증 내용
discard_proposal_is_idempotent_for_missing_id존재하지 않는 id 에 대해 discard_proposal 을 호출해도 안전한지 (idempotent). 취소 경로가 consume 된 proposal 과 실수로 두 번 호출되는 경우를 커버.
create_pczt_from_proposal_errors_for_missing_id존재하지 않는 id 로 create_pczt_from_proposal 호출 시 "Proposal not found (expired or already consumed)" 반환. Replay 시도가 깔끔하게 거부되는지 확인.
discard_proposal_after_create_pczt_failure_is_still_noop실패한 create 뒤에 discard_proposal 를 호출해도 안전한지 — Dart _send() 의 try/finally cleanup 패턴이 정당한지 확인.

Integration 테스트의 한계: Full insert → consume → replay 사이클을 테스트하려면 실제 wallet DB 와 spendable notes + 체인 tip 이 필요하다. 그건 단위 테스트 범위가 아니고 integration test 쪽에 속한다. 현재 단위 테스트는 "삭제 경로가 idempotent 하고 missing id 가 깔끔히 에러로 처리되는지" 만 커버.

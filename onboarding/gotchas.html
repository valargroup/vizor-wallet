10. 알려진 주의사항

주제주의사항
TxId byte order Rust에서 TxId::as_ref()가 원본 바이트를 반환하므로 reverse 불필요. TxId::Display는 byte-reversed hex (블록 탐색기 형식).
orchard/sapling 버전 zcash_client_sqlite가 사용하는 버전과 반드시 일치해야 함 (현재 orchard 0.11, sapling-crypto 0.5). 다르면 Rust 타입 불일치 에러.
Sapling 활성화 높이 419,200 이전 블록에서 getTreeState() 호출 시 lightwalletd가 에러 반환. 빈 hash로 ChainState::empty() 사용.
rustls CryptoProvider tonic의 tls-webpki-roots는 crypto backend를 포함하지 않음. tls-ring feature를 반드시 함께 활성화해야 rustls 0.23+에서 동작.
C callback context C function pointer는 context를 capture할 수 없다. iOS background sync에서 task.progress를 업데이트하려면 NSProgress 참조를 shared 변수에 저장하고 C callback에서 접근해야 한다.
ffi.rs 위치 ffi.rs는 api/ 밖에 위치해야 함. FRB codegen이 api/ 하위의 extern "C" 함수를 처리하려다 실패.
Sync mode 전환 timing iOS에서 mode 전환 시 이전 sync가 종료될 때까지 대기해야 한다. Swift BGTask handler는 while(mode!=2 || is_running) sleep(2s). Dart disableBackgroundSync는 while(isSyncRunning()) await 200ms.
SYNC_CANCEL 통일 FRB와 C FFI가 같은 SYNC_CANCEL AtomicBool을 공유한다. Dart cancelFullSync()와 Swift zcash_cancel_sync() 모두 동일한 flag를 설정.
BGTask expiration expirationHandler에서 mode=0으로 설정하여 resubmit 방지. 앱이 foreground로 돌아오면 onResume에서 mode==0 감지 후 foreground sync 재시작.
BATCH_SIZE 100 1000에서 100으로 줄임. cancel/mode 전환 반응이 ~10배 빨라짐. 성능 차이는 무시할 수 있는 수준 (DB write 횟수만 증가).
Debug vs Release 성능 Rust 암호학 연산이 debug 빌드에서 5-10배 느림 (opt-level=0). sync 성능 테스트는 반드시 --release로.
Wallet Creation Birthday 새 지갑 생성 시 lightwalletd에서 chain tip을 birthday로 가져옴. 네트워크 실패 시 지갑 생성 차단 (full scan 방지). DB 삭제는 birthday fetch 성공 후에만.
Scan-queue Progress 매 배치 후 suggest_scan_ranges()에서 remaining/total 블록 수를 계산하여 1.0 - remaining/total로 진행률 산출. WalletSummary::progress()는 사용하지 않음 — 노트가 없는 지갑에서 0%로 고정되는 문제 때문. scan-queue 방식은 스캔 순서와 무관하게 정확하며 FoundNote로 추가된 범위도 자동 반영.
iOS QoS BGTask handler는 using:nil로 등록, sync 작업은 .utility QoS queue에서 실행. current_thread tokio runtime이 .utility를 상속. Background에서 ~2.5x 느린 건 iOS OS throttling.
